# Beatrice Web GUI — документация

> Интерактивный редактор графов знаний в браузере. React + D3.js + FastAPI WebSocket.

Web GUI — это третий интерфейс Beatrice наряду с CLI и TUI. В отличие от терминальных версий, он даёт визуальную трёхпанельную среду: список узлов, форма редактирования и force-directed граф — всё в одном окне, с синхронизацией между панелями в реальном времени.

**Кратко:**
- Запуск одной командой: `beatrice serve graph.json`
- Открывается в браузере на `http://127.0.0.1:8576`
- Никаких серверов, БД или облачных сервисов — всё локально
- Все изменения — в памяти, сохраняются вручную (Ctrl+S)

## Установка и запуск

### Быстрый старт (production)

```bash
# Установка зависимостей
pip install -e ".[web]"
cd beatrice/web_gui/frontend && npm install && npm run build

# Запуск
beatrice serve graph.json
```

Открой браузер: [http://127.0.0.1:8576](http://127.0.0.1:8576)

### Режим разработки (с HMR)

```bash
# Терминал 1: FastAPI сервер
beatrice serve graph.json --dev

# Терминал 2: Vite HMR
cd beatrice/web_gui/frontend && npm run dev
```

Vite dev server на `http://localhost:5173` — автоматически проксирует `/ws` на FastAPI. Любые изменения в React-коде — hot reload без перезагрузки страницы.

### Makefile

```bash
make dev-web-server      # FastAPI
make dev-web-frontend    # Vite HMR
make build-web           # production сборка
make install-web         # установка зависимостей
```

## Пользовательский опыт

Интерфейс состоит из трёх панелей и тулбара. Все панели синхронизированы: выбор узла в любой панели подсвечивает его везде.

```
┌──────────────────────────────────────────────────────────┐
│  🔮 Beatrice   ●  graph.json  ● несохранено  [Save] ... │
├────────────┬─────────────────┬───────────────────────────┤
│            │                 │                           │
│  Узлы (42) │   Редактор      │    D3.js force-directed   │
│            │                 │                           │
│  [🔍 ...]  │   ID: kafka     │    ⟲ Сбросить 👻🧬↔     │
│  [тип ▼]   │   Label: ...    │    [тег ▼] [🎨]          │
│            │   Type: ...     │                           │
│  ○ Kafka   │   Desc: ...     │    ╭───╮  ╭───╮          │
│  ○ Redis   │   [Save] [✎] [🗑]│   ╱Kafka╲╱Redis╲         │
│  ○ ZK      │                 │   │     ││     │         │
│  ○ ...     │   Теги:         │   ╰─────╯╰─────╯         │
│            │   [tag1 ×] [tag2]│                           │
│            │   [+ новый...]   │    Легенда:              │
│            │                 │    ● Брокер ● БД          │
│            │   Связи:        │                           │
│            │   → ZK [исп.]   │    42 узлов · 55 рёбер    │
│            │   ← ...         │                           │
└────────────┴─────────────────┴───────────────────────────┘
```

### Тулбар

Верхняя панель содержит:

| Элемент | Назначение |
|---------|-----------|
| 🔮 Beatrice | Заголовок |
| ● / ○ | Индикатор WebSocket (зелёный = подключено, красный = нет) |
| `graph.json` | Путь к текущему файлу |
| ● несохранено | Индикатор несохранённых изменений |
| 💾 Save | Сохранить граф на диск (Ctrl+S) |
| ⟳ Reload | Перезагрузить файл с диска (сброс несохранённых изменений) |
| + Узел | Быстрое добавление нового узла через диалог |

---

### Левая панель: список узлов

Отображает все узлы графа. Каждый элемент показывает цветовую метку, label и id узла. Сироты (узлы без связей) отмечены 👻.

**Фильтры (вверху панели):**

| Элемент | Назначение |
|---------|-----------|
| 🔍 Поиск | Фильтр по id или label (case-insensitive, частичное совпадение) |
| `[Все типы ▼]` | Выпадающий список типов узлов — показывать только выбранный тип |
| `[Все теги ▼]` | Выпадающий список тегов — показывать только узлы с выбранным тегом |
| 👻 Сироты | Чекбокс — показывать только узлы без связей |

**Фильтр по конспектам:** выпадающий список с тремя значениями:
- «Все конспекты» — фильтр отключён
- «📝 С конспектом» — только узлы, у которых есть Obsidian-ссылка
- «📝 Без конспекта» — только узлы без ссылки

Все фильтры работают одновременно (AND). Результат отображается под фильтрами.

**Взаимодействие:**
- **Клик** по узлу → открывает форму редактирования в центре, подсвечивает узел на графе и делает его и его соседей непрозрачными
- Счётчик в заголовке панели показывает общее количество узлов (до фильтрации)

---

### Центральная панель: редактор

Отображается при выборе узла. Если узел не выбран — показывает заглушку «Выберите узел из списка или кликните на графе».

#### Форма атрибутов

| Поле | Тип | Назначение |
|------|-----|-----------|
| ID | text (disabled) | Идентификатор узла — нельзя изменить в форме (только через «Переименовать») |
| Метка | text | Отображаемое имя узла на графе |
| Тип | text | Категория узла (Брокер, БД, Сервис, и т.д.) — используется для раскраски |
| Описание | textarea | Произвольное текстовое описание |
| Цвет | color picker | Hex-цвет узла (на графе и в списке) |
| Размер | number (5–50) | Радиус узла на графе |

**Кнопки:**
| Кнопка | Действие |
|--------|---------|
| 💾 Сохранить | Сохраняет изменения атрибутов (активна только при наличии изменений) |
| ✎ Переименовать | Открывает диалог переименования узла (с сохранением всех связей) |
| 🗑 Удалить | Открывает диалог подтверждения удаления узла и всех его связей |

#### Конспект (Obsidian)

Ниже атрибутов узла находится поле «Конспект (Obsidian URI)». Сюда вводится ссылка на заметку в Obsidian в формате:

```
obsidian://open?vault=НАЗВАНИЕ_ХРАНИЛИЩА&file=Путь/к/Заметке
```

При наличии ссылки рядом появляется кнопка «📝 Открыть» — открывает заметку в Obsidian. На графе узлы без конспекта можно подсветить красным пунктиром через селект «📝 Без конспекта».

#### Теги

Под формой атрибутов — блок управления тегами:

- **Список тегов** узла в виде badge'й с крестиком (×) для удаления
- **Поле ввода** нового тега + кнопка «+» (Enter тоже работает)
- **Статистика тегов** всего графа: список всех тегов с количеством узлов

#### Связи

Под тегами — таблица связей выбранного узла, разделённая на входящие (←) и исходящие (→):

| Колонка | Описание |
|---------|----------|
| ID цели/источника | Идентификатор связанного узла (цветной: красный → исходящие, зелёный ← входящие) |
| Тип связи | Тип отношения (relation), курсивом |
| Метка | Label связанного узла |
| × | Кнопка удаления связи |

**Кнопка «+ Связь»** — открывает диалог добавления связи:

1. Выбрать направление: исходящая (текущий узел → ...) или входящая (... → текущий узел)
2. Выбрать узел из выпадающего списка (отображается id, label, тип)
3. Указать тип связи (необязательно, например «использует», «зависит от»)
4. Нажать «Добавить»

### Правая панель: визуализация графа

Force-directed граф на D3.js. Занимает всю правую панель.

#### Управление графом (кнопки в левом верхнем углу)

| Кнопка | Действие |
|--------|---------|
| ⟲ Сбросить | Сбросить zoom/pan, вернуть вид по умолчанию |
| 👻 Сироты | Чекбокс — скрыть/показать узлы без связей |
| ↔ Направления / ↔ Без напр. | Показать/скрыть стрелки на рёбрах |
| 🧬 Сообщества / 🧬 Типы | Переключить раскраску между Louvain-сообществами и типами узлов |
| `[— Все — ▼]` (🧬) | При активной раскраске сообществ — показать только выбранное сообщество |
| `[— Тег — ▼]` | Выбрать тег — узлы с этим тегом подсвечиваются выбранным цветом |
| 🎨 | Color picker для цвета подсветки тега |

#### Взаимодействие с графом

| Действие | Результат |
|----------|-----------|
| **Клик по узлу** | Выделяет узел, открывает форму редактирования в центре, скроллит список узлов |
| **Клик по пустому месту** | Снимает выделение |
| **Ховер по узлу** | Показывает тултип с id, label, описанием, типом, тегами |
| **Drag узла** | Перетаскивание (фиксирует позицию до двойного клика) |
| **Двойной клик по узлу** | Отпускает узел (возвращает в force-раскладку) |
| **Scroll** | Zoom |
| **Drag по фону** | Pan |
| **Клик на узле в списке** | Подсвечивает узел на графе, его соседи остаются видимыми, остальные — полупрозрачные |

#### Легенда

В левом нижнем углу — легенда раскраски. Показывает:
- При раскраске по типам: каждый тип с его цветом
- При раскраске по сообществам: каждое сообщество с его цветом

#### Информация

В правом нижнем углу: количество узлов, рёбер и сирот.

## Руководство пользователя

### Добавить узел

**Способ 1 — через тулбар:**
1. Нажать «+ Узел» в тулбаре
2. Ввести ID (обязательно), метку, тип
3. Нажать «Добавить»

**Способ 2 — через API (если есть сценарий):**
```json
{"jsonrpc": "2.0", "method": "add_node", "params": {"id": "kafka", "label": "Apache Kafka", "type": "Брокер"}, "id": 1}
```

### Редактировать узел

1. Выбрать узел в списке (слева) или кликнуть на графе (справа)
2. В центральной панели изменить поля (метка, тип, описание, цвет, размер)
3. Нажать «💾 Сохранить» (активна только при наличии изменений)

### Удалить узел

1. Выбрать узел
2. В центральной панели нажать «🗑 Удалить»
3. Подтвердить в диалоге

Вместе с узлом удаляются все его входящие и исходящие связи.

### Переименовать узел

1. Выбрать узел
2. Нажать «✎ Переименовать»
3. Ввести новый ID
4. Нажать «Переименовать»

Все связи узла сохраняются.

### Добавить связь

1. Выбрать узел
2. В блоке «Связи» нажать «+ Связь»
3. В диалоге: выбрать направление, целевой узел и тип связи
4. Нажать «Добавить»

### Удалить связь

1. Выбрать узел
2. В блоке «Связи» нажать × напротив нужной связи

### Управление тегами

**Добавить тег узлу:**
1. Выбрать узел
2. В блоке «Теги» ввести название тега в поле «Новый тег...»
3. Нажать Enter или кнопку «+»

**Удалить тег узла:**
1. Выбрать узел
2. В блоке «Теги» нажать × на badge тега

**Просмотр статистики тегов:**
Под блоком тегов выбранного узла отображается статистика всех тегов графа: каждый тег и количество узлов с ним. Обновляется автоматически при добавлении/удалении тегов.

### Поиск и фильтрация узлов

**Поиск по строке:**
- Ввести текст в поле 🔍
- Фильтруется по id и label (частичное совпадение, регистронезависимое)

**Фильтр по типу:**
- Выбрать `[тип ▼]` — отображаются только узлы выбранного типа

**Фильтр по тегу:**
- Выбрать `[тег ▼]` — отображаются только узлы с выбранным тегом

**Фильтр сирот:**
- Включить 👻 Сироты — отображаются только узлы без связей

Все фильтры работают одновременно (AND).

### Навигация по графу

- **Выбор узла:** клик на графе или в списке узлов
- **Приближение/отдаление:** scroll колёсиком мыши
- **Перемещение:** drag по пустому месту
- **Перетаскивание узлов:** drag узла
- **Сброс позиций:** двойной клик по узлу (возвращает в force-раскладку)
- **Сброс вида:** кнопка ⟲ Сбросить

### Сохранение и управление файлами

**Сохранение:**
- Нажать Ctrl+S или кнопку 💾 Save
- Изменения записываются в исходный файл
- Пока изменения не сохранены, в тулбаре горит индикатор ● несохранено

**Перезагрузка с диска:**
- Нажать ⟳ Reload
- Если есть несохранённые изменения, они будут потеряны

**Примечание:** автосохранения нет. Все изменения живут только в памяти сервера до явного сохранения.

## Техническое устройство

### Архитектура

```
┌──────────┐     WebSocket (JSON-RPC)     ┌──────────┐
│  Browser │◄────────────────────────────►│  Server   │
│  React   │     ws://host:8576/ws        │  FastAPI  │
│  SPA     │                              │  Python   │
│          │     HTTP (production)        │           │
│  D3.js   │◄── /assets/*, /* ───────────►│ NetworkX  │
│  force   │                              │ DiGraph   │
└──────────┘                              └─────┬─────┘
                                                │
                                          ┌─────▼─────┐
                                          │  graph.json │
                                          │  (JSON     │
                                          │  node-link) │
                                          └────────────┘
```

**Ключевые принципы:**
- **Транзактность:** граф читается с диска один раз при старте, все операции — в памяти, на диск пишется только по команде Save
- **Инкрементальность:** после каждого изменения сервер шлёт событие всем подключённым клиентам, клиент обновляет только изменившиеся данные
- **Нет автосохранения:** пользователь контролирует, когда изменения фиксируются на диск
- **Единый источник правды:** сервер владеет графом (NetworkX), клиент — кеш данных

### Сервер

**Файл:** `beatrice/web_gui/server.py`

**Класс `WebGraphManager`:**
Обёртка над `networkx.DiGraph`. Предоставляет методы для всех операций: CRUD узлов, связей, тегов, аналитика (Louvain, PageRank, компоненты связности, кольца BFS).

Метод `get_graph_state()` — полный дамп графа для инициализации клиента (все узлы, рёбра, статистика, Louvain-сообщества, типы, сироты).

**Функция `create_app()`:**
Создаёт FastAPI-приложение с:
- CORS (только в dev-режиме, для localhost:5173)
- `/assets/` — статика (собранный Vite)
- `/{full_path:path}` — SPA catch-all (GET → index.html)
- `/ws` — WebSocket endpoint (JSON-RPC)

**Функция `run_server()`:**
Загружает граф, создаёт приложение, запускает uvicorn.

### Протокол

WebSocket JSON-RPC 2.0.

**Запрос:**
```json
{"jsonrpc": "2.0", "method": "add_node", "params": {"id": "redis", "label": "Redis", "type": "БД"}, "id": 1}
```

**Успешный ответ:**
```json
{"jsonrpc": "2.0", "result": {"id": "redis", "label": "Redis", "type": "БД", ...}, "id": 1}
```

**Ошибка:**
```json
{"jsonrpc": "2.0", "error": {"code": -32000, "message": "Узел «redis» уже существует"}, "id": 1}
```

**Событие (broadcast всем клиентам после мутации):**
```json
{"jsonrpc": "2.0", "method": "event", "params": {"type": "node_added", "payload": {"id": "redis", ...}}}
```

#### Полный список методов API

**Узлы:**
| Метод | Параметры | Описание |
|-------|-----------|----------|
| `add_node` | `id`, `label?`, `type?`, `desc?`, `color?`, `size?`, `tags?` | Добавить узел |
| `remove_node` | `id` | Удалить узел и его связи |
| `update_node` | `id`, `label?`, `type?`, `desc?`, `color?`, `size?` | Обновить атрибуты (patch) |
| `get_node` | `id` | Получить данные узла |
| `list_nodes` | — | Список всех узлов |
| `search_nodes` | `pattern` | Поиск по id/label |
| `move_node` | `old_id`, `new_id` | Переименовать |

**Связи:**
| Метод | Параметры | Описание |
|-------|-----------|----------|
| `add_edge` | `source`, `target`, `relation?`, `weight?` | Добавить ребро |
| `remove_edge` | `source`, `target` | Удалить ребро |
| `update_edge` | `source`, `target`, `relation?`, `weight?` | Обновить атрибуты ребра |

### Управление состоянием

**WsClient** (WebSocket):
- Автоматический reconnect через 3 секунды при обрыве
- Поддержка нескольких pending-запросов (id-based)
- Таймаут запроса — 30 секунд
- Подписка на события по типу (`.on(type, handler)`)
- Метод `.call(method, params)` — Promise-based JSON-RPC вызов

**GraphStore**:
- Хранит `Map<id, NodeData>` и `EdgeData[]`
- Подписывается на все WebSocket события и обновляет локальное состояние инкрементально
- Предоставляет вычисляемые поля: `filteredNodes`, `selectedNode`, `allTypes`, `allTags`
- Методы `subscribe(listener)` для React-компонентов (forceUpdate на каждый listener при любом изменении)
- Методы-действия: `addNode`, `removeNode`, `updateNode`, `addEdge`, `removeEdge`, `addTag`, `removeTag`, `save`

**Поток данных:**
```
WsClient.on('node_added', payload) → GraphStore.nodes.set(id, data) → notify() → 
  → React компонент перерисовывается
```

### Визуализация D3.js

**Компонент:** `GraphView.tsx`

Использует `d3.forceSimulation` для force-directed раскладки:

- **Силы:** `forceLink` (рёбра, distance=150), `forceManyBody` (сила отталкивания, strength=-400), `forceCenter` (центрирование), `forceCollide` (избегание наложений, radius=size+15)
- **Zoom/Pan:** `d3.zoom` на SVG
- **Drag:** `d3.drag` на узлах (фиксация позиции при drag, отпускание при двойном клике)
- **Подсветка при выборе:** узел обводится красным, соседи остаются видимыми, остальные — opacity 0.3
- **Тултипы:** HTML-элемент, добавляемый в body при ховере, с id, label, desc, type, tags, статусом сироты
- **Раскраска (Louvain):** при активации каждый узел красится цветом сообщества из палитры (12 цветов)
- **Раскраска (тег):** выбранный тег подсвечивается выбранным цветом поверх основной раскраски
- **Стрелки:** через SVG marker-end на линиях, с возможностью скрытия
- **Легенда:** HTML-элемент, встроенный в DOM панели, обновляется при переключении раскраски
- **Фильтр сирот:** скрытие узлов с isOrphan=true

Force simulation запускается в `useEffect` и пересоздаётся при изменении графа или параметров отображения. Старая simulation останавливается при размонтировании.

### Инкрементальные события

Принцип:
1. Клиент отправляет мутирующий запрос (add_node, add_edge, ...)
2. Сервер выполняет операцию, возвращает ответ запросившему клиенту
3. Сервер шлёт событие ВСЕМ подключённым клиентам (включая запросившего)
4. Все клиенты обновляют локальное состояние по событию

Это гарантирует, что все клиенты (даже если их несколько вкладок) видят актуальное состояние без polling.

### Режим разработки

**Vite config** (`vite.config.ts`):
- Прокси `/ws` → `ws://127.0.0.1:8576` (WebSocket)
- Dev server на порту 5173

**FastAPI (dev):**
- CORS для `http://localhost:5173`
- Статика не требуется (Vite обслуживает сам)

**Production:**
- `npm run build` → `frontend/dist/`
- FastAPI раздаёт `/assets/` как StaticFiles

## Примечания

**Формат данных:** JSON node-link (nx.node_link_data). Файлы совместимы с CLI и TUI — можно открыть один и тот же файл в любом интерфейсе.

**Ограничения первой версии:**
- Один файл за сессию (переоткрыть можно через Reload, но не держать несколько одновременно)
- Нет undo/redo (есть в TUI, но не перенесён)
- Нет автосохранения
- Нет аутентификации — сервер локальный
- Нет WebGL (sigma.js) — для графов >1000 узлов может тормозить
- Нет мобильной вёрстки (но базовый responsive есть)
