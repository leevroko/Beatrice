# Планы разработки Beatrice

> Актуальные планы и направления развития.

---

- [ ] **TUI — Terminal UI для создания и редактирования графов знаний** — самый высокий приоритет. Интерактивный интерфейс в терминале: просмотр графа, добавление/удаление узлов и рёбер, поиск, подсветка сирот, работа с несколькими графами. Упрощение пользовательского опыта для тех, кто не хочет помнить CLI-команды.

- [ ] **Отказ от обёртки KnowledgeGraph** — убрать класс-обёртку из планов. CLI напрямую работает с NetworkX через JSON node-link, дополнительный уровень абстракции не нужен.

- [ ] **Интеграция с igraph / graph-tool** — для графов > 100K узлов дать возможность переключать бэкенд. Анализ (PageRank, centrality) прогонять на быстром движке, а в JSON сохранять node-link. Реализовать через фабрику: `GraphBackend("networkx")` / `GraphBackend("igraph")`.

- [ ] **Команда `graph load`** — загрузка графа из внешних форматов: GraphML, GEXF, CSV edgelist, TSV. Конвертация в нативный JSON node-link.

- [ ] **Команда `graph export`** — экспорт из нативного JSON в другие форматы: GraphML, GEXF, DOT (Graphviz), Mermaid flowchart, CSV edgelist. Сейчас экспорт есть только в `demo.py`, должен быть в CLI.

- [ ] **Команда `graph merge`** — объединение двух и более JSON-графов в один. Разрешение коллизий имён (переименование, префиксы, пропуск).

- [ ] **Команда `graph validate`** — проверка графа на целостность: битые ссылки (edge.source/target не существует), дубликаты id, изолированные компоненты.

- [ ] **Команда `graph diff`** — сравнение двух версий графа: какие узлы/рёбра добавлены, удалены, изменены. Вывод в консоль и опционально в HTML.

- [ ] **Команда `graph info`** — детальная информация об узле или ребре по id: все атрибуты, связи, метрики (degree, centrality). Расширение `neighbors`.

- [ ] **Флаг `--json` для всех команд** — вывод в JSON (в stdout) вместо человекочитаемого текста. Для интеграции с другими утилитами (`jq`, скрипты).

- [ ] **WebGL-визуализация через sigma.js** — когда граф превышает 1000 узлов, D3.js (SVG) начинает тормозить. Заменить или дополнить рендеринг на sigma.js (WebGL, 10K+ узлов плавно). Не ближайшее время.

- [ ] **Пакет на PyPI** — `pip install beatrice`. Упаковка CLI как entry_point, минимальные зависимости (только NetworkX).

- [ ] **Поддержка взвешенных мультиграфов** — несколько рёбер между одними узлами, typed edges, weighted edges с разными семантиками (вес как важность, как расстояние, как стоимость).

- [ ] **История изменений (undo/redo)** — журнал транзакций: каждая мутация графа пишется в лог, можно откатиться на N шагов назад. Формат: `journal/001-add-node.json`, `journal/002-rm-edge.json`.

- [ ] **Метрики качества графа** — density, sparsity, clustering coefficient, average degree, degree distribution, connected components. Частично уже в `stat`, но вынести в отдельную команду `graph metrics` с детализацией.

- [ ] **Плагины** — возможность писать кастомные команды на Python, которые получают NetworkX-граф и делают что угодно. `beatrice plugin run my_analysis.py graph.json`.

- [ ] **Шаблоны графов** — `beatrice init --template microservices --out graph.json` — генерация типовых графов: архитектура сервисов, дорожная карта, mindmap, org-chart.

- [ ] **Визуализация с раскраской по метрикам** — в `render` возможность раскрасить узлы по PageRank, centrality, community (Louvain) — прямо на лету, без предварительной подготовки атрибутов.
