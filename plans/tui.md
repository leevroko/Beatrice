# TUI — План реализации по итерациям

> Поэтапный план разработки TUI от нуля до полностью рабочего интерфейса.

**Всего итераций:** 9
**Принцип:** каждая итерация даёт работающий, тестируемый инкремент. Никаких «допилю потом».

---

## Итерация 0 — Фундамент

**Цель:** пустое окно TUI, которое открывает граф из файла и показывает, что он жив.

### Что делаем
- [ ] Создать структуру директории `beatrice/tui/` с файлами:
  - `__init__.py`
  - `app.py` — главный класс `BeatriceApp(Textual.App)`
  - `graph_manager.py` — обёртка над `load_graph`/`save_graph` для TUI
- [ ] Установить зависимости: `pip install textual rapidfuzz`
- [ ] Точка входа: `beatrice tui graph.json` — CLI-команда, запускающая `BeatriceApp`
- [ ] `App` загружает граф, выводит в консоль «Загружен: N узлов, M рёбер» и завершается (чтобы проверить, что граф читается)
- [ ] `--help` для `beatrice tui`

### Проверка
```bash
python3 beatrice/cli.py tui graph.json
# → TUI загружен: 9 узлов, 8 рёбер, 1 сирота
```

## Итерация 1 — Три панели (read-only)

**Цель:** три вьюпорта на экране, каждый показывает данные.

### Что делаем
- [ ] `app.py` — главный `Screen` с `Horizontal` компоновкой на три колонки
- [ ] `widgets/nodes_list.py` — `NodesList(Widget)`:
  - `ListView` с узлами (id + label + type)
  - Первый узел выбран по умолчанию
- [ ] `widgets/node_form.py` — `NodeForm(Widget)`:
  - Статические лейблы: ID, Label, Type, Description, Color, Size
  - Read-only — показывают данные текущего узла
- [ ] `widgets/links_list.py` — `LinksList(Widget)`:
  - Два заголовка: «→ Исходящие» и «← Входящие»
  - Список связей текущего узла (read-only)
- [ ] При выборе узла в `NodesList` — обновляются `NodeForm` и `LinksList`
- [ ] Стилизация:
  - Тёмная тема (TrueColor)
  - Разделители между колонками
  - Выделение активного узла

### Проверка
```bash
python3 beatrice/cli.py tui graph.json
# → Экран с тремя колонками. Слева список узлов,
#   в центре — данные первого узла, справа — его связи.
#   Можно листать список влево-вправо-вверх-вниз.
```

## Итерация 2 — Навигация (клавиатура + мышь)

**Цель:** полная навигация по TUI с клавиатуры и мыши.

### Что делаем
- [ ] Таблица хоткеев (секция 3 документа) — реализовать все:
  - **Глобальные:** `:`, `q`, `Ctrl+s`, `r`, `?`
  - **Vim-моты:** `h`/`l` — переключение вьюпортов, `j`/`k` — навигация по списку, `g`/`G` — начало/конец, `Ctrl+d`/`Ctrl+u` — полстраницы
  - **Левый вьюпорт:** `o`/`Enter` открыть, `s` — поиск, `x` — фильтр сирот, `a` — добавить, `d` — удалить
  - **Правый вьюпорт:** `o`/`Enter` открыть связанный узел, `a` — добавить связь, `d` — удалить связь, `r` — редактировать relation
- [ ] Визуальная индикация активного вьюпорта (подсветка границы/заголовка)
- [ ] Поддержка мыши:
  - Клик по узлу в списке → открыть в центре
  - Клик по связи → открыть связанный узел
  - Скролл колёсиком → прокрутка списка
- [ ] Нижняя строка с контекстными подсказками (меняется при смене вьюпорта)

### Проверка
```bash
# hjkl — навигация между и внутри вьюпортов
# Tab — цикл по элементам
# мышь — клики и скролл
# нижняя строка меняется в зависимости от активного вьюпорта
```

## Итерация 3 — Редактирование узла (NodeForm)

**Цель:** форма в центре становится редактируемой, изменения сохраняются.

### Что делаем
- [ ] `NodeForm` — заменить статические лейблы на:
  - `Input(value=..., placeholder="ID")` — id узла
  - `Input(value=..., placeholder="Label")` — label
  - `Input(value=..., placeholder="Type")` — type (с автодополнением из существующих типов)
  - `TextArea(value=..., placeholder="Description")` — description
  - `Input(value=..., placeholder="#FF0000")` — color (с превью цвета)
  - `Input(value=..., type="number")` — size
- [ ] `Tab` / `Shift+Tab` — навигация по полям формы
- [ ] `Escape` — отменить редактирование (вернуть значения как на диске)
- [ ] При изменении любого поля — помечать узел как «изменённый» (визуально)
- [ ] Автосохранение: при `Ctrl+s` или при переходе на другой узел — записать граф на диск
- [ ] `graph_manager.py` — `save()` с snapshot для undo (только сохранить, без undo UI пока)
- [ ] Статус-бар: индикатор несохранённых изменений (✎)

### Проверка
```bash
# Выбрать узел → изменить label → перейти на другой узел → 
# вернуться → label обновлён. Файл на диске изменился.
```

## Итерация 4 — Фильтрация списка узлов (NodesList)

**Цель:** левый вьюпорт с полноценной фильтрацией.

### Что делаем
- [ ] Верхняя часть `NodesList` — `Input` для fuzzy-поиска
- [ ] rapidfuzz: при вводе каждой буквы фильтровать список
  - Сравнение по `id`, `label`, `desc`, `type`
  - Отображать только совпадения, отсортированные по рейтингу
- [ ] `Escape` — сбросить поиск
- [ ] Фильтр сирот (клавиша `x`):
  - Три состояния: `ANY`, `ORPHANS`, `NON-ORPHANS`
  - Состояние отображается в заголовке списка
  - Работает совместно с fuzzy-поиском (оба фильтра применяются одновременно)
- [ ] `NodesList` генерирует `FilterChanged` сообщение

### Проверка
```bash
# Нажать s → ввести "kaf" → список показывает только Kafka и KafkaConnect
# Нажать x → список показывает только сирот
# Нажать x ещё раз → список показывает только узлы со связями
# Нажать Escape → список показывает все узлы
```

## Итерация 5 — Управление связями (LinksList)

**Цель:** правый вьюпорт с возможностью добавлять, удалять и редактировать связи.

### Что делаем
- [ ] `LinksList` — два раздела:
  - **→ Исходящие** (source = текущий узел)
  - **← Входящие** (target = текущий узел)
- [ ] Каждая строка: `узел [relation]`
- [ ] `a` — диалог добавления новой связи:
  - Выбор узла (fuzzy-поиск по всем узлам, кроме текущего)
  - Ввод relation (строка)
  - Enter — создать связь
- [ ] `d` — диалог удаления связи (с подтверждением)
- [ ] `r` — диалог редактирования relation (Input с текущим значением)
- [ ] `o`/`Enter` — открыть связанный узел (обновить центр и правый вьюпорт)
- [ ] При добавлении/удалении/редактировании — автосохранение

### Проверка
```bash
# Выбрать Kafka → правый вьюпорт показывает 4 исходящих связи
# Нажать a → выбрать Redis → relation "кэширует" → связь создана
# Нажать d на новой связи → подтвердить → связь удалена
# Нажать o на ZooKeeper → в центре теперь ZooKeeper
```

## Итерация 6 — Палитра команд

**Цель:** палитра команд с фильтрацией (список, а не vim-ввод).

### Что делаем
- [ ] `widgets/command_palette.py` — `CommandPalette(ModalScreen)`:
  - Открывается по клавише `:`
  - `Input` в верхней части
  - Список команд, фильтрующийся по мере ввода
  - Enter — выполнить выбранную команду
  - Escape — закрыть
- [ ] Команды (из документа):
  - `:q`, `:w`, `:wq`, `:e <path>`, `:search <query>`, `:add-node <id>`...
- [ ] `:undo` / `:redo` (логика snapshots уже есть, осталось привязать)
- [ ] `:help` — открыть help screen
- [ ] `:theme dark|light` — переключение темы (без реализации темы light пока)
- [ ] История команд: последние 50 команд сохраняются, доступны стрелками
- [ ] Подсказки: для каждой команды краткое описание в правой части списка

### Проверка
```bash
# Нажать : → открывается палитра → набрать "the" → в списке :theme
# Enter → тема переключена → Esc → палитра закрыта
# Нажать : → стрелка вверх → предыдущая команда
```

## Итерация 7 — Статус-бар, Help screen, Undo/Redo UI

**Цель:** оставшиеся элементы интерфейса.

### Что делаем
- [ ] **Статус-бар (верхняя строка):**
  - `📂 graph.json` — имя файла
  - Статистика: узлы, рёбра, сироты
  - Индикатор несохранённых изменений (✎)
  - Текущая тема (☀/🌙)
  - Кликабельные кнопки: `[💾]` сохранить, `[+]` добавить узел, `[🌐]` рендер HTML
- [ ] **Help screen (`?`):**
  - `widgets/help_screen.py` — `HelpScreen(ModalScreen)`
  - Таблица всех хоткеев сгруппированная по вьюпортам
  - Vim-моты, глобальные, контекстные
  - Escape — закрыть
- [ ] **Undo/Redo UI:**
  - `history.py` — класс `History` с `push(snapshot)`, `undo()`, `redo()`
  - Snapshots: полный `nx.node_link_data()` на каждое изменение
  - Лимит 25 снимков
  - `Ctrl+z` / `:undo` — откат
  - `Ctrl+y` / `:redo` — возврат
  - При undo/redo — все три вьюпорта обновляются
- [ ] **Автосохранение:**
  - При каждом изменении: add/remove/edit node, add/remove/edit edge
  - Вызов `save_graph()` + создание snapshot
  - Исключение: изменения в полях формы (сохраняются только при `Ctrl+s` или уходе с узла)

### Проверка
```bash
# Нажать ? → таблица хоткеев → Esc
# Ctrl+z → отмена последнего изменения → Ctrl+y → возврат
# Статус-бар показывает актуальную статистику
# Клик по [💾] → сохранение
```

## Итерация 8 — Темы, подтверждения, полировка

**Цель:** пользовательский опыт — гладкий и предсказуемый.

### Что делаем
- [ ] **Темы:** светлая тема (light) через `:theme light`
  - Textual CSS variables
  - Переключение на лету, без перезагрузки
- [ ] **Подтверждения (dialogs):**
  - Удаление узла: «Удалить узел «Kafka»? Все его связи тоже будут удалены. [y/N]»
  - Удаление связи: «Удалить связь Kafka → ZooKeeper? [y/N]»
  - Выход с несохранёнными: «Есть несохранённые изменения. Выйти? [y/N]»
- [ ] **Обратная связь:**
  - Сообщения в нижней строке: «Узел «Kafka» сохранён», «Связь удалена»
  - Цветовые сигналы: зелёный — успех, красный — ошибка, жёлтый — предупреждение
- [ ] **Обработка ошибок:**
  - Попытка удалить последний узел: «Нельзя удалить последний узел»
  - Попытка создать связь с несуществующим узлом: «Узел не найден»
  - Ошибка записи файла: «Не удалось сохранить файл: ...»
- [ ] **Load/Save диалоги:**
  - `:e /path/to/graph.json` — открыть другой граф (с подтверждением при несохранённых)
  - `:w` — сохранить
  - Отображение пути в статус-баре
- [ ] **Полировка:**
  - Анимации (плавное появление палитры)
  - Превью цвета в NodeForm (цветной блок рядом с Input)
  - Иконки в списке узлов: 👻 для сирот, 🏷️ для типа
  - Скорость: фильтрация через rapidfuzz не должна лагать на 1000+ узлах

### Проверка
```bash
# :theme light → светлая тема → :theme dark → обратно
# d на узле → диалог подтверждения → y → удалён
# q при несохранённых → диалог → n → остался
# :e other.json → подтверждение → открыт другой граф
```
