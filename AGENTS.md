## Project Overview

Beatrice — CLI и TUI для построения и редактирования графов знаний на базе NetworkX. Графы хранятся в формате JSON node-link, все команды работают по схеме: `прочитать → NetworkX → изменить → записать`. Никаких серверов, никаких внешних сервисов — только Python и браузер.

## Quick Start

```bash
# Install (editable, для разработки)
pip install -e .

# Install with TUI dependencies
pip install -e ".[tui]"

# Test
python3 -m pytest tests/ -v

# Run CLI
python3 -m beatrice stat graph.json

# Run CLI commands
python3 -m beatrice graph search graph.json "kafka"
python3 -m beatrice graph neighbors graph.json Kafka --direction all
python3 -m beatrice graph add-node graph.json Redis --label Redis --type "БД"
python3 -m beatrice graph edit-node graph.json Redis --desc "In-memory cache"
python3 -m beatrice graph islands graph.json
python3 -m beatrice graph louvain graph.json
python3 -m beatrice graph ring graph.json Kafka --min 2 --max 4
python3 -m beatrice graph roots graph.json
python3 -m beatrice graph frontier graph.json
python3 -m beatrice graph intersect g1.json g2.json
python3 -m beatrice graph tag add graph.json Kafka streaming
python3 -m beatrice graph tag ls graph.json
python3 -m beatrice graph render graph.json

# Run TUI
python3 -m beatrice tui graph.json
# or
beatrice-tui graph.json
```

## Architecture

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│    CLI        │     │    TUI (Textual)  │     │  Render (HTML)   │
│  (argparse)   │     │  MainScreen       │     │  D3.js template  │
│               │     │  3 panels         │     │                  │
└──────┬───────┘     └──────┬───────────┘     └────────┬─────────┘
       │                    │                          │
       └─────────┬──────────┘──────────────────────────┘
                 │
        ┌────────▼────────┐
        │   NetworkX       │
        │   DiGraph        │
        │   (nx.node_link) │
        └────────┬────────┘
                 │
        ┌────────▼────────┐
        │   JSON file      │
        │   (node-link)    │
        └─────────────────┘
```

- **CLI (`beatrice/cli.py`):** argparser с subparsers. Каждая команда — отдельная `cmd_*` функция. Все операции: `load_graph()` → мутация NetworkX → `save_graph()`.
- **TUI (`beatrice/tui/`):** Textual-based терминальный интерфейс. Три панели: NodesList (слева), NodeForm (центр), LinksList (справа). GraphManager — тонкая обёртка над NetworkX с историей snapshots (undo/redo, 25 шагов).
- **HTML Render (`beatrice/cli.py` cmd_render):** Генерирует standalone HTML с D3.js v7 force-directed графом. Весь код шаблона — в одной f-строке внутри cmd_render.
- **Формат данных:** JSON node-link (nx.node_link_data / nx.node_link_graph). Directed, not multigraph.
- **Транзактность:** Чтение → мутация в памяти → запись на диск. При ошибке диск не трогается.

## Conventions

- **Язык:** Python ≥ 3.11
- **Naming:** snake_case для функций/переменных, PascalCase для классов, UPPER_CASE для констант
- **Error handling:** Кастомное `BeatriceError` (наследует Exception). CLI-функции печатают ошибку и вызывают `sys.exit(1)`. TUI использует StatusMessage для отображения ошибок.
- **Type hints:** Аннотации типов используются (Python 3.11+). `Optional[str]` для опциональных строк.
- **Imports:** Стандартная библиотека сначала, затем сторонние (networkx, textual), затем внутренние (beatrice.*).
- **Tests:** unittest, временные файлы через tempfile, contextmanager capture_stdout для захвата вывода.
- **CLI style:** Русскоязычные сообщения для пользователя. Argparse с RawDescriptionHelpFormatter.
- **TUI style:** Textual framework. Widgets общаются через textual Message. Тёмная тема (#1a1a2e фон).
- **CSS-in-Python:** TUI виджеты используют строковые CSS-классы прямо в коде (Textual inline CSS).
- **Messages:** Все inter-widget сообщения определены в `beatrice/tui/messages.py`, наследуют `textual.message.Message`.

## Important Patterns

1. **CLI команды как отдельные функции** — каждая `cmd_*` принимает namespace от argparse, делает свою работу и завершается. Никаких классов-обёрток.
2. **GraphManager + Snapshot history** — Undo/redo через snapshots полного состояния графа (nx.node_link_data). Лимит 25 снимков. При каждом изменении — `_push_snapshot()`.
3. **Textual inter-widget messaging** — Виджеты не вызывают методы друг друга напрямую. Всё через `post_message()`/обработчики `on_*`.
4. **Модальные диалоги как Screen** — Каждый диалог (`ConfirmDialog`, `InputDialog`, `AddLinkDialog`) — отдельный `ModalScreen`. Результат возвращается через `dismiss()`.
5. **Fuzzy-поиск через rapidfuzz** — Во всех списках (узлы, команды, выбор узла для связи) используется `fuzz.partial_ratio()` с порогом 30-40.
6. **D3.js HTML шаблон** — Весь генератор HTML — это одна f-строка в `cmd_render()`. Тёмная/светлая тема через подстановку цветов.
7. **Транзактность** — `load_graph()` → мутация → `save_graph()`. При ошибке чтения/записи — `BeatriceError`, файл не повреждается.
8. **Автосохранение счётчика** — GraphManager хранит `_counter` для генерации ID новых узлов. Счётчик сохраняется в `G.graph["beatrice_counter"]` при записи.
